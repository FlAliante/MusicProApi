Diccionario de API para MusicPro

Método

Es la forma en que una API permite realizar alguna acción dentro de un sistema.

Ejemplo:

fetch('https://api.com/api/get_productos', {
  method: 'GET'
})
.then(response => response.json())
.then(data => console.log(data))
.catch(error => console.error(error));
Parámetros:
  • method: El método HTTP utilizado para la solicitud (p. ej., GET, POST).
  • headers: Cabeceras adicionales para incluir en la solicitud HTTP.
  • body: Los datos a enviar con la solicitud, si corresponde.

Endpoint

Es la URL única que corresponde a un recurso en particular.

Ejemplo:

https://api.com/api/get_productos

Autenticación

Las APIs pueden requerir algún tipo de autenticación para acceder a la funcionalidad que ofrecen.

Tipo de autenticación:
  • Token: Proporciona un token de acceso con cada solicitud (p. ej., JWT).
  • Clave API: Proporciona una clave de API única con cada solicitud.
  • Básica: Utiliza la autenticación básica de HTTP (nombre de usuario y contraseña).

Respuesta

Cuando se realiza una solicitud a una API, se recibe una respuesta en formato JSON u otro formato.

Formatos admitidos:
  • JSON: Un formato de datos ligero y fácil de leer y escribir para humanos y máquinas.
  • XML: Un formato de marcas similar al HTML utilizado para describir datos.
  • Texto sin formato: Puede ser útil para ciertas solicitudes, como devolver información binaria.
